*你可能想提前 Google 的概念：中断、CPU 寄存器*

**目标：让我们之前沉默的引导扇区打印一些文本**

我们将改进我们的无限循环引导扇区，并在屏幕上打印一些内容。我们将为此引发一个中断。

在这个例子中，我们将把"Hello"单词的每个字符写入寄存器 `al`（`ax` 的低位部分），将字节 `0x0e` 写入 `ah`（`ax` 的高位部分），并引发中断 `0x10`，这是视频服务的通用中断。

`ah` 中的 `0x0e` 告诉视频中断，我们想要运行的实际函数是"以 tty 模式写入 `al` 的内容"。

虽然我们只会设置一次 tty 模式，但在现实世界中我们无法确定 `ah` 的内容是恒定的。当我们睡眠时，其他进程可能在 CPU 上运行，没有正确清理并在 `ah` 中留下垃圾数据。

对于这个例子，我们不需要担心这一点，因为我们是 CPU 上唯一运行的东西。

我们的新引导扇区看起来像这样：
```nasm
mov ah, 0x0e ; tty 模式
mov al, 'H'
int 0x10
mov al, 'e'
int 0x10
mov al, 'l'
int 0x10
int 0x10 ; 'l' 仍然在 al 中，记得吗？
mov al, 'o'
int 0x10

jmp $ ; 跳转到当前地址 = 无限循环

; 填充和魔术数字
times 510 - ($-$$) db 0
dw 0xaa55
```

你可以使用 `xxd file.bin` 检查二进制数据

无论如何，你知道该怎么做：

`nasm -fbin boot_sect_hello.asm -o boot_sect_hello.bin`

`qemu boot_sect_hello.bin`

你的引导扇区将显示'Hello'并挂在无限循环上
