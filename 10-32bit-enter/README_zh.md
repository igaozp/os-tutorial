*你可能想提前 Google 的概念：中断、流水线*

**目标：进入 32 位保护模式并测试我们之前课程的代码**

要跳入 32 位模式：

1. 禁用中断
2. 加载我们的 GDT
3. 在 CPU 控制寄存器 `cr0` 上设置一个位
4. 通过发出精心设计的远跳转来刷新 CPU 流水线
5. 更新所有段寄存器
6. 更新栈
7. 调用一个众所周知的标签，其中包含 32 位中的第一个有用代码

我们将把这个过程封装在文件 `32bit-switch.asm` 中。打开它并查看代码。

进入 32 位模式后，我们将调用 `BEGIN_PM`，这是我们实际有用代码（例如内核代码等）的入口点。你可以在 `32bit-main.asm` 中阅读代码。编译并运行这最后一个文件，你将在屏幕上看到两条消息。

恭喜！我们的下一步将是编写一个简单的内核
