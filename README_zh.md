操作系统教程
===========

_⚠️ 注意！这是一个旧的、已废弃的项目，存在技术和设计方面的问题 [详见此处](https://github.com/cfenollosa/os-tutorial/issues/269)。请尽情享受本教程，但如果您想学习操作系统设计，请寻找更现代和权威的资源。⚠️_

如何从零开始创建一个操作系统！

我一直想学习如何从零开始制作一个操作系统。在大学里，我学习了如何实现高级特性（分页、信号量、内存管理等），但是：

- 我从未从自己的引导扇区开始
- 大学课程很难，所以我不记得大部分内容了
- 我受够了那些认为阅读现有内核（即使很小）是学习操作系统的好方法的人

受到[这份文档](http://www.cs.bham.ac.uk/~exr/lectures/opsys/10_11/lectures/os-dev.pdf)
和 [OSDev wiki](http://wiki.osdev.org/) 的启发，我将尝试制作简短的分步 README 和
代码示例供任何人学习。老实说，本教程基本上就是第一份文档，但分成了更小的部分，并且没有理论。

更新：更多资源：[操作系统开发小册子](https://littleosbook.github.io)，
[JamesM 的内核开发教程](https://web.archive.org/web/20160412174753/http://www.jamesmolloy.co.uk/tutorial_html/index.html)


特点
--------

- 本课程是一个代码教程，面向熟悉底层计算的人群。例如，对操作系统工作原理有好奇心但没有时间或意愿从头到尾阅读 Linux 内核的程序员。
- 理论内容很少。是的，这是一个特点。Google 就是你的理论讲师。一旦你大学毕业，过多的理论比没有理论更糟糕，因为它会让事情看起来比实际更困难。
- 课程很短，可能需要 5-15 分钟完成。相信我，相信你自己。你可以做到！


如何使用本教程
------------------------

1. 从第一个文件夹开始，按顺序向下学习。它们建立在之前的代码之上，所以如果你直接跳到文件夹 05 而不知道为什么有一个 `mov ah, 0x0e`，那是因为你错过了第 02 课。
真的，按顺序来。你总是可以跳过你已经知道的内容。

2. 打开 README 并阅读第一行，其中详细说明了在阅读代码之前应该熟悉的概念。Google 你不熟悉的概念。第二行说明了每节课的目标。
阅读它们，因为它们解释了我们为什么要这样做。"为什么"和"如何"一样重要。

3. 阅读 README 的其余部分。它**非常简洁**。

4. （可选）在阅读 README 后，尝试自己编写代码文件。

5. 查看代码示例。它们有非常详细的注释。

6. （可选）尝试实验并尝试破坏它们。确保你理解某事的唯一方法是尝试破坏它或用不同的命令复制它。


简而言之：首先阅读每个文件夹中的 README，然后阅读代码文件。如果你足够勇敢，尝试自己编写它们。
