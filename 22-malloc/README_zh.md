*你可能想提前 Google 的概念：malloc*

**目标：实现一个内存分配器**

我们将向 `libc/mem.c` 添加一个内核内存分配器。它被实现为一个指向空闲内存的简单指针，它不断增长。

`kmalloc()` 函数可用于请求对齐的页面，它还将返回真实的物理地址，以供以后使用。

我们将更改 `kernel.c`，将所有"shell"代码留在那里，让我们只尝试新的 `kmalloc()`，并检查我们的第一个页面从 0x10000 开始（在 `mem.c` 上硬编码），后续的 `kmalloc()` 产生一个新地址，该地址与前一个地址对齐 4096 字节或 0x1000。

请注意，我们添加了一个新的 `strings.c:hex_to_ascii()` 以更好地打印十六进制数字。

另一个装饰性修改是将 `types.c` 重命名为 `type.c` 以保持语言一致性。

其余文件与上一课相同，未更改。
