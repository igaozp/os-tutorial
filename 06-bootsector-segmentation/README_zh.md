*你可能想提前 Google 的概念：分段*

**目标：学习如何使用 16 位实模式分段来寻址内存**

如果你对分段感到舒适，跳过这一课。

我们在第 3 课中使用 `[org]` 进行了分段。分段意味着你可以为你引用的所有数据指定一个偏移量。

这是通过使用特殊寄存器来完成的：`cs`、`ds`、`ss` 和 `es`，分别用于代码、数据、栈和额外（即用户定义）

注意：它们被 CPU *隐式*使用，所以一旦你为 `ds` 设置了某个值，那么你的所有内存访问都将被 `ds` 偏移。
[在此处阅读更多](http://wiki.osdev.org/Segmentation)

此外，为了计算真实地址，我们不只是连接两个地址，而是*重叠*它们：`segment << 4 + address`。例如，如果 `ds` 是 `0x4d`，那么 `[0x20]` 实际上指的是 `0x4d0 + 0x20 = 0x4f0`

理论够了。看看代码，稍微玩一下。

提示：我们不能将字面量 `mov` 到这些寄存器，我们必须先使用一个通用寄存器。
